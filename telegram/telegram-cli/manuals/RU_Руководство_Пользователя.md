# telegram-cli — Руководство пользователя

Это руководство объясняет, как установить и использовать **telegram-cli** — инструмент командной строки, который позволяет создавать резервные копии сообщений Telegram, читать и управлять ими без официального приложения. Вам не нужно быть программистом. Просто следуйте шагам по порядку.

---

Telegram — один из самых популярных мессенджеров в мире: более 900 миллионов активных пользователей в месяц в десятках языков и стран.

| Язык | ISO 639‑1 | Основные страны | Примерное число пользователей Telegram |
|------|-----------|-----------------|----------------------------------------|
| Русский | ru | Россия, Украина, Беларусь | ≈ 120–150 миллионов |
| Английский | en | Индия, США, Великобритания | ≈ 110–130 миллионов |
| Персидский (фарси) | fa | Иран, Афганистан, Таджикистан | ≈ 40–50 миллионов |
| Турецкий | tr | Турция, Кипр, Германия (диаспора) | ≈ 25–35 миллионов |
| Арабский | ar | Египет, Ирак, Саудовская Аравия | ≈ 20–30 миллионов |
| Немецкий | de | Германия, Австрия, Швейцария | ≈ 8–12 миллионов |

В некоторых странах или регионах Telegram может быть недоступен или заблокирован. В таких случаях telegram-cli поможет — он позволяет работать с данными Telegram из командной строки, независимо от официальных приложений.

Инструмент особенно полезен для **автономной обработки** содержимого Telegram — например, для резервного копирования переписок, групп и каналов с последующим анализом с помощью инструментов ИИ или другой автоматизации.

Вам не нужно быть программистом, чтобы пользоваться этим инструментом. Просто следуйте приведённым ниже шагам.

## Быстрый старт — Windows

> Этот раздел предназначен для **пользователей Windows**, которые скачали `telegram-cli.exe`. Если вы используете macOS или вариант на Python, перейдите к разделу [Контрольный список настройки](#контрольный-список-настройки).

Вот пять команд, которые нужны для создания первой резервной копии. Каждая команда подробно объяснена далее в этом руководстве.

**Прежде чем начать:** Скачайте `telegram-cli.exe` и это руководство пользователя, как описано в разделе [Установка](#установка). Затем поместите `telegram-cli.exe` в папку по вашему выбору — например, `C:\Users\ВашеИмя\telegram-cli\`. Откройте командную строку в этой папке (см. [Как открыть командную строку](#как-открыть-командную-строку-windows-или-терминал-macos) ниже).

**Шаг 1 — Получите учётные данные API** на <https://my.telegram.org/apps> (один раз; подробнее см. [Шаг 2](#шаг-2--создание-приложения-telegram)).

**Шаг 2 — Войдите в систему:**
```
telegram-cli init
```
Введите `api_id`, `api_hash` и номер телефона по запросу. Введите код входа, отправленный в приложение Telegram.

**Шаг 3 — Проверьте сессию:**
```
telegram-cli whoami
```
Вы должны увидеть своё имя и номер телефона.

**Шаг 4 — Получите список переписок:**
```
telegram-cli get-dialogs
```
Найдите ID переписки, которую хотите сохранить (число в столбце `ID`).

**Шаг 5 — Создайте резервную копию:**
```
telegram-cli backup -1001234567890 --limit=500
```
Замените `-1001234567890` на реальный ID из Шага 4. Сообщения сохраняются в папку `synchromessotron\` в текущем каталоге.

> **Предупреждение SmartScreen при первом запуске:** Windows может показать _«Windows защитила ваш компьютер»_. Нажмите **Подробнее** → **Всё равно выполнить**. Это происходит только один раз.

## Обзор — Что умеет telegram-cli?

| Команда | Что делает |
|---------|-----------|
| `init` | Настройка учётных данных и сессии |
| `whoami` | Проверка сессии и отображение информации о пользователе |
| `ping` | Проверка доступности Telegram |
| `get-dialogs` | Список переписок |
| `backup` | Полное или инкрементальное резервное копирование сообщений |
| `send` | Отправка сообщения в переписку |
| `edit` | Редактирование ранее отправленного сообщения |
| `delete` | Удаление собственных сообщений |
| `download-media` | Загрузка фото, видео или файла из сообщения |
| `help` | Справка на вашем языке |
| `version` | Информация о версии |

> **Что такое «диалог»?** В Telegram «диалог» — это любая переписка: личный чат с человеком, групповой чат или канал.

## Установка

### Шаг 1 — Выберите вариант и скачайте файл

Перейдите на страницу [Releases](https://github.com/vsirotin/synchromessotron/releases/), найдите последнюю стабильную версию, откройте раздел «Assets» и скачайте файл для вашей платформы:

| Платформа | Файл | Дополнительное ПО |
|-----------|------|-------------------|
| **Windows** | `telegram-cli.exe` | Не требуется |
| **macOS** | `telegram-cli-macos.zip` | Не требуется |
| **Любая (Python)** | `telegram-cli.pyz` | Да — Python ≥ 3.11 |

Поместите скачанный файл в папку по вашему выбору — например, папку `telegram-cli` на рабочем столе или в домашнем каталоге.

> **Какой вариант выбрать?**
>
> - **Windows или macOS** — используйте файл для своей платформы. Дополнительное ПО не требуется.
> - **Вариант на Python** — выберите его, если хотите проверить код перед запуском: `.pyz` — стандартный Python zip-архив, который можно открыть и прочитать. Также работает на Linux.

**Примечания по первому запуску на разных платформах:**

- **Windows:** При первом запуске SmartScreen может показать _«Windows защитила ваш компьютер»_. Нажмите **Подробнее** → **Всё равно выполнить**. Это происходит только один раз.
- **macOS:** После распаковки откройте Terminal в папке с файлом и выполните эти две команды один раз:
  ```
  chmod +x telegram-cli
  xattr -d com.apple.quarantine telegram-cli
  ```
  Это снимает карантин macOS и делает файл исполняемым.
- **Вариант на Python:** Требуется Python 3.11 или новее. Проверьте командой `python3 --version`. При необходимости установите Python с сайта <https://www.python.org/downloads/>.

### Шаг 2 — Создание приложения Telegram

Для использования telegram-cli вам нужны собственные учётные данные Telegram API. Это разовая настройка — повторять её не потребуется.

1. Откройте <https://my.telegram.org/apps> в браузере и войдите с помощью номера телефона Telegram.
2. Нажмите **API development tools**.
3. Заполните форму **Create new application**. Название приложения (App title) и короткое имя (Short name) могут быть любыми, например `synchromessotron` / `syncbot`. Платформу можно оставить `Other`.

   ![Диалог создания нового приложения](../docs/images/telegram1.png)

4. Нажмите **Create application**.
5. Вы увидите **App api_id** (короткое число) и **App api_hash** (длинная строка букв и цифр). Оставьте эту вкладку браузера открытой — оба значения понадобятся на следующем шаге.

   ![api_id и api_hash](../docs/images/telegram2.png)

### Как открыть командную строку (Windows) или терминал (macOS)

Вам понадобится окно командной строки (Windows) или Terminal (macOS), открытое в папке, куда вы сохранили файл telegram-cli.

> **Windows — открыть командную строку в нужной папке:**
>
> 1. Откройте **Проводник** и перейдите в папку, куда вы сохранили `telegram-cli.exe`.
> 2. Нажмите на адресную строку вверху (в ней отображается путь к папке).
> 3. Введите `cmd` и нажмите **Enter**. Откроется чёрное окно командной строки, уже находящееся в этой папке.

> **macOS — открыть Terminal в нужной папке:**
>
> 1. Откройте **Finder** и перейдите в папку, куда вы сохранили `telegram-cli`.
> 2. Щёлкните правой кнопкой мыши (или Control-click) по папке.
> 3. Выберите **New Terminal at Folder**. Откроется окно Terminal в этой папке.
>
> (Если этот пункт не отображается: перейдите в **Системные настройки → Конфиденциальность и безопасность → Расширения → Расширения Finder** и включите **New Terminal at Folder**.)

### Шаг 3 — Настройка сессии

Откройте командную строку (Windows) или Terminal (macOS) в папке с файлом (см. выше) и выполните:

| Платформа | Команда |
|-----------|---------|
| **Windows** | `telegram-cli init` |
| **macOS** | `./telegram-cli init` |
| **Python** | `python3 telegram-cli.pyz init` |

Команда выполнит следующее:
1. Запросит **api_id**, **api_hash** и **номер телефона** (из Шага 2).
2. Отправит код входа в ваше приложение Telegram — введите его по запросу.
3. Если у вас включена двухэтапная аутентификация, запросит **пароль 2FA**.
4. Создаст файл `config.yaml` с вашими учётными данными и сессией.

> **Что такое пароль 2FA?** Это пароль, *который вы сами задали* при включении двухэтапной аутентификации в Telegram. Чтобы проверить или изменить его: откройте Telegram → **Настройки → Конфиденциальность и безопасность → Двухэтапная аутентификация**.

**Ожидаемый результат:**

```
✓ Session created and saved to config.yaml
  Run 'telegram-cli whoami' to verify.
```

### Шаг 4 — Проверка настройки

| Платформа | Команда |
|-----------|---------|
| **Windows** | `telegram-cli whoami` |
| **macOS** | `./telegram-cli whoami` |
| **Python** | `python3 telegram-cli.pyz whoami` |

**Ожидаемый результат:**

```
✓ Session valid
  User ID:   123456789
  Name:      Your Name
  Username:  @yourhandle
  Phone:     +1234567890
```

Если вместо этого отображается ошибка, обратитесь к разделу [Обработка ошибок](#обработка-ошибок) ниже.

Также убедитесь, что Telegram доступен:

| Платформа | Команда |
|-----------|---------|
| **Windows** | `telegram-cli ping` |
| **macOS** | `./telegram-cli ping` |
| **Python** | `python3 telegram-cli.pyz ping` |

**Ожидаемый результат:**

```
✓ Telegram is reachable (42.3 ms)
```

## Безопасность — Защита учётных данных

Файл `config.yaml`, создаваемый telegram-cli, содержит учётные данные Telegram API и вашу сессию. **Любой, кто получит этот файл, сможет получить доступ к вашему аккаунту Telegram.**

**Правила:**

1. **Никогда не передавайте `config.yaml` другим.** Относитесь к нему как к паролю.
2. **Никогда не загружайте его в интернет** — не размещайте на GitHub, Google Drive, Dropbox, не отправляйте по электронной почте и не сохраняйте в общедоступных местах.
3. **Храните его только в той же папке, что и файл telegram-cli** на вашем компьютере.
4. **Если вы подозреваете, что кто-то мог увидеть ваш `config.yaml`** — немедленно отзовите сессию в Telegram: перейдите в **Настройки → Устройства** (или **Настройки → Конфиденциальность и безопасность → Активные сессии**) и завершите подозрительную сессию. Затем снова выполните `telegram-cli init` для создания новой.

> Файл `config.yaml.example` — шаблон без реальных учётных данных, его можно передавать другим.

## Где сохраняются файлы

Все данные, создаваемые telegram-cli (резервные копии, скачанные медиафайлы), хранятся в **папке вывода**.

### Расположение по умолчанию

По умолчанию в папке, из которой запускается telegram-cli, создаётся папка `synchromessotron`:

```
<текущая папка>/synchromessotron/
```

### Выбор другой папки

Другое расположение можно указать с помощью `--outdir`:

| Платформа | Пример |
|-----------|--------|
| **Windows** | `telegram-cli backup -1001234567890 --outdir=C:\MyBackups` |
| **macOS** | `./telegram-cli backup -1001234567890 --outdir=/Users/yourname/MyBackups` |
| **Python** | `python3 telegram-cli.pyz backup -1001234567890 --outdir=/path/to/my/data` |

> **Правило конфликта:** Если одновременно заданы `--outdir` и `output_dir` в `config.yaml` и они указывают на разные пути, команда завершится с ошибкой. Удалите одно из значений для устранения конфликта.

Перед началом работы telegram-cli проверяет, доступна ли папка для записи. Большинство папок на рабочем столе или в домашнем каталоге уже имеют права на запись. Если возникает ошибка прав записи, см. раздел [Решение проблем](#решение-проблем).

### Как организованы переписки

Для каждой переписки создаётся подпапка по следующему правилу именования:

```
<первые 10 символов имени>_<ID переписки>
```

- Пробелы в имени заменяются символом подчёркивания (`_`).
- Если имя короче 10 символов, используется полное имя.

| Переписка | Подпапка |
|-----------|----------|
| `Мемуары кочевого программиста. Байки, были, думы` (ID -718738386) | `Мемуары_ко_718738386` |
| `Telegram` (ID 777000) | `Telegram_777000` |

Внутри каждой папки переписки сообщения организованы по годам, а при большом количестве — по месяцам или дням (подробнее см. раздел [Дополнительно](#дополнительно)).

### Какие файлы сохраняются

На каждом уровне создаются два файла:

| Файл | Содержимое |
|------|-----------|
| `messages.json` | Полные данные сообщений (для использования с другими инструментами). |
| `messages.md` | Автор, дата и текст сообщения — удобно читать в любом текстовом редакторе. |

### Сохранение фото, видео и другого контента

По умолчанию сохраняются только сообщения. Для включения другого контента добавьте флаги к команде `backup`:

| Флаг | Что сохраняется |
|------|----------------|
| `--media` | Фото и видео |
| `--files` | Документы и вложения |
| `--music` | Аудиодорожки |
| `--voice` | Голосовые сообщения |
| `--links` | Превью ссылок и URL |
| `--gifs` | GIF-анимации |
| `--members` | Список участников переписки |

Пример — резервная копия сообщений и фото:

| Платформа | Команда |
|-----------|---------|
| **Windows** | `telegram-cli backup -1001234567890 --media` |
| **macOS** | `./telegram-cli backup -1001234567890 --media` |
| **Python** | `python3 telegram-cli.pyz backup -1001234567890 --media` |

Пример — резервная копия всего содержимого:

```
telegram-cli backup -1001234567890 --media --files --music --voice --links --gifs --members
```

### Пример структуры папок

После выполнения `backup` с флагами `--media --members`:

```
synchromessotron/
├── Мемуары_ко_718738386/
│   ├── members/
│   ├── 2025/
│   │   ├── 01/
│   │   │   ├── messages.json
│   │   │   ├── messages.md
│   │   │   └── media/
│   │   └── 02/
│   │       ├── messages.json
│   │       └── messages.md
│   └── 2026/
│       ├── messages.json
│       └── messages.md
└── Telegram_777000/
    └── 2026/
        ├── messages.json
        └── messages.md
```

В этом примере:
- В переписке «Мемуары кочевого…» в 2025 году было более 50 сообщений, поэтому они разбиты по месяцам. В 2026 году сообщений меньше — они остались в годовой папке.
- В «Telegram» сообщений мало — разбивка по месяцам не нужна.

## Справочник команд

### get-dialogs — Список переписок

```
get-dialogs [--limit=N] [--outdir=DIR]
```

**Параметры:**

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `--limit` | 100 | Максимальное число возвращаемых переписок. Используйте `--limit=500` или больше для полного списка. |
| `--outdir` | — | Также сохранить `dialogs.json` в эту папку (см. [Где сохраняются файлы](#где-сохраняются-файлы)). |

**Примеры:**

| Платформа | Команда |
|-----------|---------|
| **Windows** | `telegram-cli get-dialogs --limit=50` |
| **macOS** | `./telegram-cli get-dialogs --limit=50` |
| **Python** | `python3 telegram-cli.pyz get-dialogs --limit=50` |

**Ожидаемый результат:**

```
  TYPE         ID                 NAME
  ------------ ------------------ ------------------------------------------
  User         123456789          Your Name  @yourhandle
  Channel      -1001234567890     Family Group  @familygroup
  Chat         -987654321         Old Project

3 dialogs found.
```

При указании `--outdir` таблица по-прежнему выводится на экран, а `dialogs.json` также сохраняется в папку вывода.

При ошибке: `NETWORK_ERROR`, `PERMISSION_DENIED`, `RATE_LIMITED`, `SESSION_INVALID`. См. раздел [Обработка ошибок](#обработка-ошибок).

### backup — Резервное копирование сообщений

```
backup <dialog_id> [--since=TIMESTAMP] [--upto=TIMESTAMP] [--limit=N] [--outdir=DIR]
       [--media] [--files] [--music] [--voice] [--links] [--gifs] [--members]
       [--estimate] [--count]
```

**Параметры:**

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `<dialog_id>` | — | ID переписки (обязательный). Используйте `get-dialogs` для поиска. |
| `--since` | — | Загружать только сообщения, отправленные **после** указанной даты и времени (формат: `2026-01-01T00:00:00+00:00`). |
| `--upto` | — | Загружать только сообщения, отправленные **не позднее** указанной даты и времени. |
| `--limit` | 100 | Максимальное число загружаемых сообщений. |
| `--outdir` | `./synchromessotron` | Сохранять файлы в эту папку вместо папки по умолчанию. |
| `--media` | выкл. | Также загружать фото и видео. |
| `--files` | выкл. | Также загружать документы и вложения. |
| `--music` | выкл. | Также загружать аудиодорожки. |
| `--voice` | выкл. | Также загружать голосовые сообщения. |
| `--links` | выкл. | Также сохранять превью ссылок и URL. |
| `--gifs` | выкл. | Также загружать GIF-анимации. |
| `--members` | выкл. | Также сохранять список участников переписки. |
| `--estimate` | выкл. | Показать ожидаемое время резервного копирования и остановиться. Ничего не загружается. |
| `--count` | выкл. | Показать количество сообщений и файлов и остановиться. Ничего не загружается. |

**Примеры:**

| Платформа | Команда |
|-----------|---------|
| **Windows** | `telegram-cli backup -1001234567890 --limit=500` |
| **macOS** | `./telegram-cli backup -1001234567890 --limit=500` |
| **Python** | `python3 telegram-cli.pyz backup -1001234567890 --limit=500` |

**Ожидаемый результат** (только сообщения):

```
✓ 500 messages saved to synchromessotron/Telegram_777000/2026/
```

**Ожидаемый результат** (с флагами `--media --files`):

```
✓ 500 messages saved to synchromessotron/Telegram_777000/2026/
✓ 23 media files downloaded
✓ 7 documents downloaded
```

**Инкрементальное копирование** — только сообщения после указанной даты:

```
telegram-cli backup -1001234567890 --since=2026-03-01
```

**Ожидаемый результат:**

```
✓ 12 messages saved to synchromessotron/Telegram_777000/2026/03/
```

**Временной диапазон** — сообщения между двумя датами (`--since` должна быть раньше `--upto`):

```
telegram-cli backup -1001234567890 ^
    --since=2026-01-01T00 ^
    --upto=2026-01-01T10
```

> **Распространённая ошибка:** Если `--since` указана *после* `--upto`, будет сохранено 0 сообщений, поскольку ни одно сообщение не может быть одновременно позже февраля и раньше января. Всегда проверяйте, что `--since` — более ранняя дата, чем `--upto`.

**Оценка времени перед большим копированием:**

```
telegram-cli backup -1001234567890 --limit=5000 --estimate
```

**Ожидаемый результат:**

```
≈ 12 minutes (5000 messages, estimated 2.4 ms per message)
```

Оценка приблизительная — реальное время зависит от скорости интернета и ограничений Telegram. Ничего не загружается.

**Подсчёт сообщений перед загрузкой:**

```
telegram-cli backup -1001234567890 --count
```

**Ожидаемый результат:**

```
Messages: 350 total
  photo: 42
  link/webpage: 8
  video: 5
  file/document: 2
```

При ошибке: `ENTITY_NOT_FOUND`, `NETWORK_ERROR`, `PERMISSION_DENIED`, `RATE_LIMITED`. См. раздел [Обработка ошибок](#обработка-ошибок).

## Обработка ошибок

При возникновении проблемы telegram-cli выводит сообщение с описанием ошибки и рекомендуемыми действиями.

Пример:

```
Error [RATE_LIMITED]: Too many requests — retry after 30s
  retry_after: 30
```

**Значение каждой ошибки и способ её устранения:**

| Ошибка | Что произошло | Что делать |
|--------|--------------|-----------|
| `AUTH_FAILED` | Ваш аккаунт Telegram деактивирован или заблокирован. | Обратитесь в службу поддержки Telegram. |
| `ENTITY_NOT_FOUND` | Указанный ID переписки или сообщения не существует. | Используйте `get-dialogs` для поиска правильного ID. |
| `INTERNAL_ERROR` | Внутри telegram-cli произошла непредвиденная ошибка. | Сообщите о проблеме, приложив полный текст ошибки. |
| `NETWORK_ERROR` | telegram-cli не может подключиться к Telegram. | Проверьте подключение к интернету или попробуйте позже. |
| `NOT_MODIFIED` | Вы попытались изменить сообщение, но новый текст совпадает со старым. | Используйте другой текст. |
| `PERMISSION_DENIED` | У вас нет прав на чтение или запись в этой переписке. | Проверьте права доступа для данной переписки. |
| `RATE_LIMITED` | Вы отправили слишком много запросов за короткое время; Telegram просит подождать. | Подождите указанное в сообщении об ошибке количество секунд и повторите попытку. |
| `SESSION_INVALID` | Ваша сессия истекла или была отозвана. | Снова выполните `telegram-cli init` для повторного входа. |

## Коды завершения

По окончании работы telegram-cli возвращает операционной системе числовой код (полезно для скриптов):

| Код | Значение |
|-----|---------|
| 0 | Всё выполнено успешно. |
| 1 | Использована неверная команда или параметр. |
| 2 | Произошла ошибка Telegram (см. таблицу выше). |

## Дополнительно

### Управление глубиной папок с помощью `--split_threshold`

По умолчанию сообщения группируются в годовые папки. Если в одном году накапливается много сообщений, telegram-cli автоматически создаёт ежемесячные подпапки, затем ежедневные, ежечасовые и, при необходимости, поминутные.

Порог, при котором создаётся более глубокий уровень, задаётся параметром `--split_threshold` (по умолчанию: `100`). Например, при запуске резервного копирования с `--split_threshold=20` новый уровень подпапок создаётся всякий раз, когда в папке накапливается более 20 сообщений.

```
telegram-cli backup -1001234567890 --split_threshold=20
```

Это полезно для более детальной организации очень активных переписок. Для большинства пользователей значение по умолчанию является оптимальным.

Глубина следует этому порядку:

```
year/ → month/ → day/ → hour/ → minute/
```

Сообщения всегда хранятся на самом глубоком уровне — никогда не разделяясь между двумя уровнями.

## Решение проблем

### telegram-cli не может подключиться к Telegram

Если отображается ошибка `NETWORK_ERROR`, Telegram недоступен с вашего компьютера.

**Попробуйте следующие шаги по порядку:**

1. Убедитесь, что интернет работает (откройте сайт в браузере).
2. Повторите попытку через несколько минут — у Telegram может быть временный сбой.
3. Если Telegram заблокирован в вашем регионе, рассмотрите использование VPN.
4. Выполните `telegram-cli ping`, чтобы определить, проблема в соединении или в сессии.

---

### Ошибка прав на запись

Если telegram-cli сообщает, что не может записывать в папку вывода, у вашей учётной записи нет прав на создание файлов в этом месте.

**Простейшее решение для большинства пользователей:** Запускайте telegram-cli из домашней папки или с рабочего стола, либо используйте `--outdir` и укажите папку в вашем домашнем каталоге. Там права на запись есть всегда.

**Windows — проверка и предоставление прав на запись:**

Щёлкните правой кнопкой мыши по целевой папке → **Свойства** → вкладка **Безопасность** → убедитесь, что у вашего пользователя есть право **Запись**. Если нет, нажмите **Изменить**, выберите своего пользователя и отметьте флажок **Запись**.

Для опытных пользователей то же самое можно сделать в командной строке:

```
icacls "C:\path\to\folder"
```

Найдите `(W)` или `(F)` рядом с вашим именем пользователя. Чтобы предоставить право на запись:

```
icacls "C:\path\to\folder" /grant %USERNAME%:W
```

**macOS / Linux — проверка и предоставление прав на запись:**

Большинство папок в вашем домашнем каталоге (Рабочий стол, Документы, Загрузки) уже имеют права на запись. Для пользовательского пути проверьте командой:

```
ls -ld /path/to/folder
```

Если в правах доступа отсутствует `w`, предоставьте его командой:

```
chmod u+w /path/to/folder
```

---

### Код входа не приходит

Если после выполнения `telegram-cli init` вы не получили код входа:

1. Убедитесь, что введённый номер телефона указан правильно и включает код страны (например, `+79161234567`).
2. Проверьте приложение Telegram — код приходит как обычное сообщение от официального аккаунта Telegram.
3. Подождите минуту и снова выполните `telegram-cli init`.

---

### Сессия истекла

Если отображается ошибка `SESSION_INVALID`, срок вашей сессии истёк или она была отозвана (например, если вы завершили все активные сессии в настройках Telegram).

Выполните `telegram-cli init` для повторного входа.
