# telegram-cli — راهنمای کاربر

این راهنما توضیح می‌دهد که چگونه **telegram-cli** را نصب کرده و از آن استفاده کنید — ابزاری خط فرمان که به شما امکان می‌دهد پیام‌های تلگرامتان را بدون نیاز به اپلیکیشن رسمی پشتیبان‌گیری کنید، بخوانید و مدیریت کنید. نیازی به برنامه‌نویس بودن ندارید. فقط مراحل را به ترتیب دنبال کنید.

---

تلگرام یکی از پرکاربردترین پیام‌رسان‌های جهان است که بیش از ۹۰۰ میلیون کاربر فعال ماهانه در ده‌ها زبان و کشور دارد.

| زبان | ISO 639‑1 | کشورهای برتر | تخمین کاربران تلگرام |
|------|-----------|--------------|----------------------|
| روسی | ru | روسیه، اوکراین، بلاروس | ≈ ۱۲۰–۱۵۰ میلیون |
| انگلیسی | en | هند، آمریکا، بریتانیا | ≈ ۱۱۰–۱۳۰ میلیون |
| فارسی | fa | ایران، افغانستان، تاجیکستان | ≈ ۴۰–۵۰ میلیون |
| ترکی | tr | ترکیه، قبرس، آلمان (مهاجران) | ≈ ۲۵–۳۵ میلیون |
| عربی | ar | مصر، عراق، عربستان سعودی | ≈ ۲۰–۳۰ میلیون |
| آلمانی | de | آلمان، اتریش، سوئیس | ≈ ۸–۱۲ میلیون |

تلگرام گاهی در برخی کشورها یا مناطق جغرافیایی در دسترس نیست یا مسدود می‌شود. در این موارد، telegram-cli می‌تواند کمک کند — این ابزار به شما اجازه می‌دهد مستقل از اپلیکیشن‌های رسمی، با داده‌های تلگرام خود از خط فرمان کار کنید.

این ابزار به ویژه برای **پردازش آفلاین** محتوای تلگرام مفید است — برای مثال، پشتیبان‌گیری از مکالمات، گروه‌ها و کانال‌ها برای تحلیل بعدی با ابزارهای هوش مصنوعی یا سایر اتوماسیون‌ها.

برای استفاده از این ابزار نیازی به برنامه‌نویس بودن ندارید. فقط مراحل زیر را دنبال کنید.

## شروع سریع — ویندوز

> این بخش برای **کاربران ویندوز** است که فایل `telegram-cli.exe` را دانلود کرده‌اند. اگر از macOS استفاده می‌کنید یا نسخه Python را دارید، به [فهرست مراحل راه‌اندازی](#فهرست-مراحل-راهاندازی) بروید.

پنج دستوری که باید اجرا کنید تا اولین پشتیبان‌گیری را انجام دهید. هر دستور در ادامه این راهنما به تفصیل توضیح داده شده است.

**پیش از شروع:** فایل `telegram-cli.exe` را در پوشه‌ای دلخواه قرار دهید — مثلاً `C:\Users\YourName\telegram-cli\`. سپس Command Prompt را در همان پوشه باز کنید (روش کار را در [نحوه باز کردن Command Prompt](#نحوه-باز-کردن-command-prompt-ویندوز-یا-terminal-macos) ببینید).

**مرحله ۱ — دریافت اطلاعات API** از <https://my.telegram.org/apps> (یک‌بار انجام می‌شود؛ جزئیات در [مرحله ۲](#مرحله-۲--ساخت-اپلیکیشن-تلگرام)).

**مرحله ۲ — ورود:**
```
telegram-cli init
```
وقتی از شما خواسته شد، `api_id`، `api_hash` و شماره تلفن خود را وارد کنید. کد ورودی ارسال‌شده به اپ تلگرامتان را تایپ کنید.

**مرحله ۳ — تأیید:**
```
telegram-cli whoami
```
باید نام و شماره تلفن خود را ببینید.

**مرحله ۴ — فهرست مکالمات:**
```
telegram-cli get-dialogs
```
شناسه مکالمه‌ای را که می‌خواهید پشتیبان بگیرید پیدا کنید (عدد ستون `ID`).

**مرحله ۵ — پشتیبان‌گیری:**
```
telegram-cli backup -1001234567890 --limit=500
```
عدد `-1001234567890` را با شناسه واقعی از مرحله ۴ جایگزین کنید. پیام‌های شما در پوشه `synchromessotron\` در مسیر جاری ذخیره می‌شوند.

> **هشدار SmartScreen در اولین اجرا:** ویندوز ممکن است پیام _«Windows protected your PC»_ را نمایش دهد. روی **More info** کلیک کنید ← سپس **Run anyway**. این اتفاق فقط یک بار می‌افتد.

## معرفی — telegram-cli چه کاری می‌کند؟

| دستور | عملکرد |
|-------|--------|
| `init` | راه‌اندازی اطلاعات ورود و نشست |
| `whoami` | تأیید نشست و نمایش اطلاعات کاربر |
| `ping` | بررسی دسترس‌پذیری تلگرام |
| `get-dialogs` | فهرست مکالمات شما |
| `backup` | پشتیبان‌گیری کامل یا تدریجی از پیام‌ها |
| `send` | ارسال پیام به یک مکالمه |
| `edit` | ویرایش پیام ارسال‌شده قبلی |
| `delete` | حذف پیام‌های خودتان |
| `download-media` | دانلود عکس، ویدیو یا فایل از یک پیام |
| `help` | نمایش راهنما به زبان دلخواه |
| `version` | نمایش اطلاعات نسخه |

> **«dialog» چیست؟** در تلگرام، «dialog» به هر مکالمه‌ای گفته می‌شود — چت خصوصی با یک نفر، گروه یا کانال.

## نصب

### مرحله ۱ — انتخاب نسخه و دانلود

به صفحه [Releases](https://github.com/vsirotin/synchromessotron/releases/) بروید، آخرین نسخه پایدار را پیدا کنید، «Assets» را باز کنید و فایل مناسب پلتفرم خود را دانلود کنید:

| پلتفرم | فایل | نرم‌افزار اضافی لازم است؟ |
|--------|------|--------------------------|
| **Windows** | `telegram-cli.exe` | خیر |
| **macOS** | `telegram-cli-macos.zip` | خیر |
| **هر سیستمی (Python)** | `telegram-cli.pyz` | بله — Python نسخه ۳.۱۱ یا بالاتر |

فایل دانلود‌شده را در پوشه‌ای دلخواه قرار دهید — مثلاً پوشه `telegram-cli` روی دسکتاپ یا در پوشه کاربری شما.

> **کدام نسخه را انتخاب کنم؟**
>
> - **Windows یا macOS** — از فایل مخصوص پلتفرم استفاده کنید. نیازی به نرم‌افزار اضافی نیست.
> - **نسخه Python** — این را انتخاب کنید اگر می‌خواهید قبل از اجرا کد را بررسی کنید: فایل `.pyz` یک آرشیو zip استاندارد Python است که می‌توانید باز کرده و بخوانید. روی Linux هم کار می‌کند.

**نکات اولین اجرا بر اساس پلتفرم:**

- **Windows:** اولین بار که فایل را اجرا می‌کنید، SmartScreen ممکن است پیام _«Windows protected your PC»_ را نشان دهد. روی **More info** کلیک کنید ← سپس **Run anyway**. این اتفاق فقط یک بار می‌افتد.
- **macOS:** پس از از حالت فشرده درآوردن، Terminal را در پوشه باز کنید و این دو دستور را یک‌بار اجرا کنید:
  ```
  chmod +x telegram-cli
  xattr -d com.apple.quarantine telegram-cli
  ```
  این دستورات قرنطینه دانلود macOS را حذف کرده و فایل را قابل اجرا می‌کنند.
- **نسخه Python:** به Python نسخه ۳.۱۱ یا جدیدتر نیاز دارد. با `python3 --version` بررسی کنید. در صورت نیاز، Python را از <https://www.python.org/downloads/> نصب کنید.

### مرحله ۲ — ساخت اپلیکیشن تلگرام

برای استفاده از telegram-cli به اطلاعات API تلگرام خودتان نیاز دارید. این کار فقط یک بار انجام می‌شود و هرگز لازم نیست تکرار شود.

1. آدرس <https://my.telegram.org/apps> را در مرورگر باز کنید و با شماره تلفن تلگرامتان وارد شوید.
2. روی **API development tools** کلیک کنید.
3. فرم **Create new application** را پر کنید. عنوان اپ و نام کوتاه می‌توانند هر چیزی باشند، مثلاً `synchromessotron` / `syncbot`. پلتفرم را می‌توانید روی `Other` بگذارید.

   ![دیالوگ ساخت اپلیکیشن جدید](../docs/images/telegram1.png)

4. روی **Create application** کلیک کنید.
5. **App api_id** (یک عدد کوتاه) و **App api_hash** (یک رشته بلند از حروف و اعداد) خود را خواهید دید. این تب مرورگر را باز نگه دارید — در مرحله بعد به هر دو مقدار نیاز خواهید داشت.

   ![api_id و api_hash](../docs/images/telegram2.png)

### نحوه باز کردن Command Prompt (ویندوز) یا Terminal (macOS)

باید یک پنجره Command Prompt (ویندوز) یا Terminal (macOS) در پوشه‌ای که فایل telegram-cli را ذخیره کرده‌اید باز کنید.

> **ویندوز — باز کردن Command Prompt در پوشه:**
>
> 1. **File Explorer** را باز کنید و به پوشه‌ای که `telegram-cli.exe` را ذخیره کرده‌اید بروید.
> 2. روی نوار آدرس در بالا کلیک کنید (مسیر پوشه را نشان می‌دهد).
> 3. `cmd` را تایپ کنید و **Enter** بزنید. یک پنجره سیاه Command Prompt در همان پوشه باز می‌شود.

> **macOS — باز کردن Terminal در پوشه:**
>
> 1. **Finder** را باز کنید و به پوشه‌ای که `telegram-cli` را ذخیره کرده‌اید بروید.
> 2. روی پوشه راست‌کلیک کنید (یا Control-کلیک کنید).
> 3. **New Terminal at Folder** را انتخاب کنید. یک پنجره Terminal در آن پوشه باز می‌شود.
>
> (اگر این گزینه را نمی‌بینید: به **System Settings → Privacy & Security → Extensions → Finder Extensions** بروید و **New Terminal at Folder** را فعال کنید.)

### مرحله ۳ — راه‌اندازی نشست

Command Prompt (ویندوز) یا Terminal (macOS) را در پوشه‌ای که فایل را قرار داده‌اید باز کنید (روش بالا) و این دستور را اجرا کنید:

| پلتفرم | دستور |
|--------|-------|
| **Windows** | `telegram-cli init` |
| **macOS** | `./telegram-cli init` |
| **Python** | `python3 telegram-cli.pyz init` |

این دستور:
1. **api_id**، **api_hash** و **شماره تلفن** شما را می‌خواهد (از مرحله ۲).
2. یک کد ورود به اپ تلگرام شما ارسال می‌کند — وقتی خواسته شد آن را وارد کنید.
3. اگر تأیید دو مرحله‌ای فعال باشد، **رمز عبور 2FA** شما را می‌خواهد.
4. یک فایل `config.yaml` با اطلاعات ورود و نشست شما می‌سازد.

> **رمز عبور 2FA چیست؟** رمزی است که *خودتان* هنگام فعال کردن تأیید دو مرحله‌ای در تلگرام تنظیم کرده‌اید. برای بررسی یا تغییر آن: تلگرام را باز کنید ← **Settings → Privacy and Security → Two-Step Verification**.

**نتیجه مورد انتظار:**

```
✓ Session created and saved to config.yaml
  Run 'telegram-cli whoami' to verify.
```

### مرحله ۴ — تأیید راه‌اندازی

| پلتفرم | دستور |
|--------|-------|
| **Windows** | `telegram-cli whoami` |
| **macOS** | `./telegram-cli whoami` |
| **Python** | `python3 telegram-cli.pyz whoami` |

**نتیجه مورد انتظار:**

```

  User ID:   123456789
  Name:      Your Name

```

اگر به جای این خطا دیدید، بخش [مدیریت خطاها](#مدیریت-خطاها) را بررسی کنید.

همچنین از دسترس‌پذیری تلگرام اطمینان حاصل کنید:

| پلتفرم | دستور |
|--------|-------|
| **Windows** | `telegram-cli ping` |
| **macOS** | `./telegram-cli ping` |
| **Python** | `python3 telegram-cli.pyz ping` |

**نتیجه مورد انتظار:**

```
✓ Telegram is reachable (42.3 ms)
```

## امنیت — محافظت از اطلاعات ورود

فایل `config.yaml` که telegram-cli می‌سازد حاوی اطلاعات API تلگرام و نشست ورود شماست. **هر کسی که این فایل را داشته باشد می‌تواند به حساب تلگرام شما دسترسی پیدا کند.**

**قوانین:**

1. **هرگز `config.yaml` را با کسی به اشتراک نگذارید.** با آن مثل رمز عبور رفتار کنید.
2. **هرگز آن را در اینترنت آپلود نکنید** — در GitHub، Google Drive، Dropbox، ایمیل یا هیچ مکان مشترکی قرار ندهید.
3. **فقط در همان پوشه فایل telegram-cli** روی کامپیوتر خودتان نگه دارید.
4. **اگر فکر می‌کنید کسی `config.yaml` شما را دیده است** — فوراً نشست را در تلگرام باطل کنید: به **Settings → Devices** (یا **Settings → Privacy and Security → Active Sessions**) بروید و نشست مشکوک را پایان دهید. سپس `telegram-cli init` را دوباره اجرا کنید تا نشست جدیدی بسازید.

> فایل `config.yaml.example` یک قالب بدون اطلاعات واقعی است — به اشتراک گذاشتن آن مشکلی ندارد.

## محل ذخیره فایل‌ها

تمام داده‌های تولیدشده توسط telegram-cli (پشتیبان‌ها، رسانه‌های دانلودشده) در یک **پوشه خروجی** ذخیره می‌شوند.

### مکان پیش‌فرض

به طور پیش‌فرض، پوشه‌ای به نام `synchromessotron` در داخل پوشه‌ای که telegram-cli را از آن اجرا می‌کنید ایجاد می‌شود:

```
<پوشه جاری>/synchromessotron/
```

### انتخاب پوشه دیگر

می‌توانید با `--outdir` مکان دیگری انتخاب کنید:

| پلتفرم | مثال |
|--------|------|
| **Windows** | `telegram-cli backup -1001234567890 --outdir=C:\MyBackups` |
| **macOS** | `./telegram-cli backup -1001234567890 --outdir=/Users/yourname/MyBackups` |
| **Python** | `python3 telegram-cli.pyz backup -1001234567890 --outdir=/path/to/my/data` |

> **قانون تعارض:** اگر هم `--outdir` و هم `output_dir` در `config.yaml` تنظیم شده باشند و به مسیرهای متفاوتی اشاره کنند، دستور با خطا متوقف می‌شود. برای رفع تعارض یکی از آن‌ها را حذف کنید.

telegram-cli قبل از شروع بررسی می‌کند که آیا می‌تواند در پوشه بنویسد یا نه. اغلب مکان‌های روی دسکتاپ یا در پوشه کاربری شما مجوز نوشتن دارند. اگر خطای «write permission» گرفتید، بخش [عیب‌یابی](#عیبیابی) را ببینید.

### سازماندهی مکالمات

برای هر مکالمه، یک زیرپوشه با این قانون نام‌گذاری ایجاد می‌شود:

```
<۱۰ کاراکتر اول نام>_<شناسه مکالمه>
```

- فاصله‌ها در نام با زیرخط (`_`) جایگزین می‌شوند.
- اگر نام کمتر از ۱۰ کاراکتر باشد، نام کامل استفاده می‌شود.

| مکالمه | زیرپوشه |
|--------|---------|
| `Мемуары кочевого программиста. Байки, были, думы` (شناسه ‎-718738386) | `Мемуары_ко_718738386` |
| `Telegram` (شناسه 777000) | `Telegram_777000` |

در داخل هر پوشه مکالمه، پیام‌ها بر اساس سال سازماندهی می‌شوند و در صورت تعداد زیاد پیام، بر اساس ماه یا روز تقسیم می‌شوند (جزئیات در [پیشرفته](#پیشرفته)).

### فایل‌های ذخیره‌شده

در هر سطح، دو فایل ایجاد می‌شود:

| فایل | محتوا |
|------|-------|
| `messages.json` | داده کامل پیام‌ها (برای استفاده با ابزارهای دیگر). |
| `messages.md` | نویسنده، تاریخ و متن پیام — به راحتی در هر ویرایشگر متنی خوانده می‌شود. |

### ذخیره عکس‌ها، ویدیوها و محتوای دیگر

به طور پیش‌فرض فقط پیام‌ها ذخیره می‌شوند. برای گنجاندن محتوای دیگر، پرچم‌هایی به دستور `backup` اضافه کنید:

| پرچم | چه چیزی ذخیره می‌شود |
|------|--------------------|
| `--media` | عکس‌ها و ویدیوها |
| `--files` | اسناد و پیوست‌های فایل |
| `--music` | آهنگ‌های صوتی |

### backup — پشتیبان‌گیری از پیام‌ها

```
backup <dialog_id> [--since=TIMESTAMP] [--upto=TIMESTAMP] [--limit=N] [--outdir=DIR]
       [--media] [--files] [--music] [--voice] [--links] [--gifs] [--members]
       [--estimate] [--count]
```

**پارامترها:**

| پارامتر | پیش‌فرض | توضیح |
|---------|---------|-------|
| `<dialog_id>` | — | شناسه مکالمه (الزامی). برای پیدا کردن آن از `get-dialogs` استفاده کنید. |
| `--since` | — | فقط پیام‌های ارسال‌شده **پس از** این تاریخ و ساعت را دانلود کنید (قالب: `2026-01-01T00:00:00+00:00`). |
| `--upto` | — | فقط پیام‌های ارسال‌شده **تا و در** این تاریخ و ساعت را دانلود کنید. |
| `--limit` | 100 | حداکثر تعداد پیام‌های دانلودشده. |
| `--outdir` | `./synchromessotron` | فایل‌ها را به جای مکان پیش‌فرض در این پوشه ذخیره کنید. |
| `--media` | خاموش | عکس‌ها و ویدیوها را هم دانلود کنید. |
| `--files` | خاموش | اسناد و پیوست‌های فایل را هم دانلود کنید. |
| `--music` | خاموش | آهنگ‌های صوتی را هم دانلود کنید. |
| `--voice` | خاموش | پیام‌های صوتی را هم دانلود کنید. |
| `--links` | خاموش | پیش‌نمایش لینک‌ها و URLها را هم ذخیره کنید. |
| `--gifs` | خاموش | انیمیشن‌های GIF را هم دانلود کنید. |
| `--members` | خاموش | فهرست شرکت‌کنندگان مکالمه را هم ذخیره کنید. |
| `--estimate` | خاموش | نشان دهید پشتیبان‌گیری چقدر طول می‌کشد، سپس متوقف شود. چیزی دانلود نمی‌شود. |
| `--count` | خاموش | نشان دهید چند پیام و فایل وجود دارد، سپس متوقف شود. چیزی دانلود نمی‌شود. |

**مثال‌ها:**

| پلتفرم | دستور |
|--------|-------|
| **Windows** | `telegram-cli backup -1001234567890 --limit=500` |
| **macOS** | `./telegram-cli backup -1001234567890 --limit=500` |
| **Python** | `python3 telegram-cli.pyz backup -1001234567890 --limit=500` |

**نتیجه مورد انتظار** (فقط پیام‌ها):

```
✓ 500 messages saved to synchromessotron/Telegram_777000/2026/
```

**نتیجه مورد انتظار** (با `--media --files`):

```
✓ 500 messages saved to synchromessotron/Telegram_777000/2026/
✓ 23 media files downloaded
✓ 7 documents downloaded
```

**پشتیبان‌گیری تدریجی** — فقط پیام‌های پس از یک تاریخ:

```
telegram-cli backup -1001234567890 --since=2026-03-01T00:00:00+00:00
```

**نتیجه مورد انتظار:**

```
✓ 12 messages saved to synchromessotron/Telegram_777000/2026/03/
```

**بازه زمانی** — پیام‌های بین دو تاریخ (`--since` باید زودتر از `--upto` باشد):

```
telegram-cli backup -1001234567890 ^
    --since=2026-01-01T00:00:00+00:00 ^
    --upto=2026-01-31T23:59:59+00:00
```

> **اشتباه رایج:** تنظیم `--since` به تاریخی *بعد از* `--upto` منجر به ذخیره ۰ پیام می‌شود، چون هیچ پیامی نمی‌تواند هم بعد از فوریه و هم قبل از ژانویه باشد. همیشه بررسی کنید که `--since` تاریخ زودتری نسبت به `--upto` باشد.

**تخمین زمان قبل از پشتیبان‌گیری بزرگ:**

```

```

**نتیجه مورد انتظار:**

```
≈ 12 minutes (5000 messages, estimated 2.4 ms per message)
```

تخمین تقریبی است — زمان واقعی به سرعت اینترنت و محدودیت‌های تلگرام بستگی دارد. چیزی دانلود نمی‌شود.

  video: 5

## مدیریت خطاها

هنگام بروز مشکل، telegram-cli پیامی چاپ می‌کند که مشکل و راه‌حل را توضیح می‌دهد.

مثال:

```
Error [RATE_LIMITED]: Too many requests — retry after 30s
  retry_after: 30
```

**معنای هر خطا و اقدام لازم:**

| خطا | چه اتفاقی افتاده | چه کاری باید کرد |
|-----|-----------------|-----------------|
| `AUTH_FAILED` | حساب تلگرام شما غیرفعال یا مسدود شده است. | با پشتیبانی تلگرام تماس بگیرید. |
| `ENTITY_NOT_FOUND` | شناسه مکالمه یا پیامی که استفاده کردید وجود ندارد. | از `get-dialogs` برای پیدا کردن شناسه درست استفاده کنید. |
| `INTERNAL_ERROR` | یک خطای غیرمنتظره درون telegram-cli رخ داده است. | مشکل را گزارش دهید و پیام خطای کامل را ضمیمه کنید. |
| `NETWORK_ERROR` | telegram-cli نمی‌تواند به تلگرام دسترسی داشته باشد. | اتصال اینترنت را بررسی کنید یا بعداً دوباره امتحان کنید. |
| `NOT_MODIFIED` | سعی کردید پیامی را ویرایش کنید، اما متن جدید مثل متن قبلی است. | از متن متفاوتی استفاده کنید. |
| `PERMISSION_DENIED` | مجوز خواندن از این مکالمه یا نوشتن در آن را ندارید. | حقوق دسترسی خود به آن مکالمه را بررسی کنید. |
| `RATE_LIMITED` | در مدت کوتاهی درخواست‌های زیادی ارسال کردید؛ تلگرام از شما می‌خواهد صبر کنید. | به اندازه ثانیه‌هایی که در پیام خطا نشان داده شده صبر کنید، سپس دوباره امتحان کنید. |
| `SESSION_INVALID` | نشست ورود شما منقضی شده یا باطل شده است. | دستور `telegram-cli init` را دوباره اجرا کنید تا مجدداً وارد شوید. |

وقتی telegram-cli پایان می‌یابد، یک عدد به سیستم‌عامل برمی‌گرداند (مفید برای اسکریپت‌ها):

| کد | معنا |
|----|------|
| 0 | همه چیز درست کار کرد. |
| 1 | دستور یا پارامتر اشتباهی استفاده کردید. |
| 2 | یک خطای تلگرام رخ داد (جدول بالا را ببینید). |

## پیشرفته

### کنترل عمق پوشه‌بندی با `--split_threshold`

به طور پیش‌فرض، پیام‌ها در پوشه‌های سالانه گروه‌بندی می‌شوند. اگر یک سال پیام‌های زیادی داشته باشد، telegram-cli به طور خودکار زیرپوشه‌های ماهانه، سپس روزانه، ساعتی و در نهایت دقیقه‌ای ایجاد می‌کند.

آستانه‌ای که تقسیم عمیق‌تر را فعال می‌کند با `--split_threshold` (پیش‌فرض: `100`) کنترل می‌شود. به عنوان مثال، اگر پشتیبان‌گیری را با `--split_threshold=20` اجرا کنید، هر بار که یک پوشه بیش از ۲۰ پیام داشته باشد یک سطح زیرپوشه جدید ایجاد می‌شود.

```
telegram-cli backup -1001234567890 --split_threshold=20
```

این گزینه برای سازماندهی دقیق‌تر مکالمات بسیار فعال مفید است. برای اکثر کاربران مقدار پیش‌فرض کافی است.

ترتیب عمق به این صورت است:

```
year/ → month/ → day/ → hour/ → minute/
```

پیام‌ها همیشه در عمیق‌ترین سطح ذخیره می‌شوند — هرگز بین دو سطح تقسیم نمی‌شوند.

## عیب‌یابی

### telegram-cli نمی‌تواند به تلگرام دسترسی داشته باشد

اگر `NETWORK_ERROR` می‌بینید، تلگرام از کامپیوتر شما قابل دسترسی نیست.

**این مراحل را به ترتیب امتحان کنید:**

1. بررسی کنید که اتصال اینترنت شما کار می‌کند (یک وب‌سایت را در مرورگر باز کنید).
2. چند دقیقه بعد دوباره امتحان کنید — ممکن است تلگرام دچار اختلال موقت شده باشد.
3. اگر تلگرام در منطقه شما مسدود است، استفاده از VPN را در نظر بگیرید.
4. دستور `telegram-cli ping` را اجرا کنید تا مشخص شود مشکل از اتصال است یا از نشست.

### خطای مجوز نوشتن

اگر telegram-cli بگوید نمی‌تواند در پوشه خروجی بنویسد، حساب کاربری شما مجوز ایجاد فایل در آنجا را ندارد.

**ساده‌ترین راه‌حل برای اکثر کاربران:** telegram-cli را از پوشه کاربری یا دسکتاپ اجرا کنید، یا از `--outdir` برای اشاره به پوشه‌ای در مسیر کاربری استفاده کنید. این مکان‌ها همیشه مجوز نوشتن دارند.

**ویندوز — بررسی و اعطای مجوز نوشتن:**

روی پوشه هدف راست‌کلیک کنید ← **Properties** ← تب **Security** ← بررسی کنید که کاربر شما مجوز **Write** داشته باشد. اگر نداشت، روی **Edit** کلیک کنید، کاربر خود را انتخاب کنید و تیک **Write** را بزنید.

برای کاربران پیشرفته، همین کار را می‌توان در Command Prompt انجام داد:

```
icacls "C:\path\to\folder"
```

به دنبال `(W)` یا `(F)` کنار نام کاربری خود بگردید. برای اعطای مجوز نوشتن:

```
icacls "C:\path\to\folder" /grant %USERNAME%:W
```

**macOS / Linux — بررسی و اعطای مجوز نوشتن:**

اکثر پوشه‌های موجود در مسیر کاربری (دسکتاپ، Documents، Downloads) از قبل مجوز نوشتن دارند. اگر از مسیر سفارشی استفاده می‌کنید، با این دستور بررسی کنید:

```
ls -ld /path/to/folder
```

اگر `w` در مجوزها وجود نداشت، با این دستور آن را اعطا کنید:

```
chmod u+w /path/to/folder
```

### کد ورود دریافت نمی‌شود

اگر پس از اجرای `telegram-cli init` کد ورود دریافت نکردید:

1. مطمئن شوید شماره تلفنی که وارد کردید صحیح است و شامل کد کشور می‌شود (مثلاً `+98912345678`).
2. اپ تلگرام خود را بررسی کنید — کد به عنوان یک پیام معمولی از حساب رسمی تلگرام ارسال می‌شود.
3. یک دقیقه صبر کنید و دستور `telegram-cli init` را دوباره اجرا کنید.

---

### نشست منقضی شده

اگر `SESSION_INVALID` می‌بینید، ورود شما منقضی شده یا باطل شده است (مثلاً اگر تمام نشست‌های فعال را در تنظیمات تلگرام پایان داده‌اید).

دستور `telegram-cli init` را اجرا کنید تا دوباره وارد شوید.
